# ECD Source Hierarchy and Canonization Workflow v0.1

Status: Project Reference

Purpose: Defines the working source hierarchy for Eismark canon documents and lists recovered development ideas by review category against the current ECD.

## 1. ECD Master Index - Source Map

The ECD Master Index is the table of contents for the project. It tells the project where everything lives and what its canon status is.

Use it to locate:

- ECD entries.
- Audit records.
- Development entries.
- Archive records.
- Resolved canon evolutions.
- Open review areas.

The Master Index is not the full canon text. It is the map of the canon system.

## 2. ECD Archive - Current Canon

The ECD Archive is the current source of truth for Eismark.

If anything from an older conversation, recovered chat, previous PDF, development note, or non-canon backlog conflicts with the ECD Archive, the ECD Archive wins unless the topic has been explicitly reopened.

Use the ECD Archive to determine:

- Current locked canon.
- Under-review canon-adjacent material.
- Existing entry language.
- Cross-entry relationships.
- Which older concepts have been replaced or preserved.

Do not silently overwrite ECD canon with recovered material. Retcons, promotions, or reopenings require an explicit decision.

## 3. New Canonization Ideas - Development Backlog

The New Canonization Ideas document is not canon.

It is a curated archive of recovered conversations containing ideas that were close to being canonized. Treat it as a pool of candidate lore to compare against the ECD Archive.

Use the backlog to find:

- Strong unrecovered ideas.
- Older versions of current canon.
- Candidate NPCs, factions, locations, laws, and campaign concepts.
- Development history that explains how canon evolved.

Do not treat the backlog as a canon authority. Its purpose is intake, comparison, and future promotion review.

## Review Categories

When reviewing backlog ideas, sort them into these categories.

### Already Canon

The idea is already incorporated into the ECD Archive or the readable guide.

Use this when the backlog preserves an older wording of something that has already become canon.

### Should be Promoted

The idea is excellent and not yet present in canon.

Use this when the concept fits current ECD canon and deserves a new or expanded ECD entry, pending explicit approval.

### Needs Revision

The concept is good, but outdated, incomplete, duplicated, or changed by later lore.

Use this when an idea should be rewritten before promotion.

### Archived / Superseded

The idea has been replaced by newer canon, duplicated by another backlog entry, or preserved only as development history.

Use this when the idea is worth keeping for reference but should not be promoted.

### Contradicts Canon

The idea conflicts with the current ECD Archive.

Use this when the idea should remain non-canon unless the user intentionally retcons or reopens the topic.

## Working Rule

The New Canonization Ideas document is an intake document, not a canon authority.

When in doubt:

1. Check the ECD Master Index to find the relevant entries.
2. Read the ECD Archive for current canon.
3. Compare the backlog idea against the ECD.
4. Classify the idea before proposing any promotion.
5. Ask for explicit approval before changing canon.
